# Prisma Standards

Version: 1.0

Status: Frozen

# Philosophy

Prisma is the only interface between Oja and PostgreSQL.

Every database operation must pass through Prisma.

Direct SQL is prohibited unless absolutely necessary.

# Schema

The Prisma schema is the single source of truth for the database.

Schema changes require migrations.

Never edit production databases manually.

# Models

Every model represents one database table.

Model names

PascalCase

Examples

User

Store

Product

Order

Payment

Field names

camelCase

Database mapping

snake_case

# IDs

All primary keys use UUID.

Generated by Prisma.

# Relationships

Always define explicit relationships.

Use foreign keys.

Avoid implicit relationships.

# Queries

Keep queries simple.

Use pagination for collections.

Select only required fields.

Avoid unnecessary eager loading.

# Transactions

Use transactions when

Creating orders

Updating inventory

Completing payments

Activating subscriptions

Operations must be atomic.

# Soft Deletes

Preferred where business history must remain.

Financial records are never physically deleted.

# Indexes

Index only

Foreign keys

Frequently searched fields

Sorting fields

Avoid unnecessary indexes.

# Migrations

Every schema change requires a migration.

Migration names should describe the change.

Example

add-product-variants

create-subscriptions

# Seeding

Seed only development data.

Production data is never seeded.

# Rules

Never bypass Prisma.

Never expose Prisma directly to controllers.

Repositories own all Prisma access.
