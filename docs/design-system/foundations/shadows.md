# Shadows

Version: 1.0

Status: Frozen

# Philosophy

Shadows communicate elevation.

They should be subtle.

Oja avoids dramatic floating elements.

The interface should feel grounded.

# Elevation Levels

Level 0

No Shadow

Usage

Page backgrounds

Flat sections

---

Level 1

Small Shadow

Usage

Cards

Buttons

Inputs

Dropdowns

---

Level 2

Medium Shadow

Usage

Popovers

Sticky headers

Floating toolbars

---

Level 3

Large Shadow

Usage

Modals

Side panels

Command palette

# Hover Elevation

Interactive components increase by one elevation level on hover.

The transition should be subtle.

# Active State

Pressed components reduce elevation.

This provides tactile feedback.

# Focus State

Focus should be indicated primarily through

Border

Outline

Ring

Do not rely solely on shadows.

# Rules

Do not stack multiple shadows.

Avoid colored shadows.

Shadows should never become the primary visual element.

Use elevation only when it improves hierarchy.
