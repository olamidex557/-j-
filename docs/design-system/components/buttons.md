# Buttons

Version: 1.0

Status: Frozen

# Philosophy

Buttons represent actions.

Every button should clearly communicate what will happen when clicked.

Avoid ambiguous labels.

# Button Hierarchy

Primary

Highest emphasis.

Used once per screen.

Examples

Create Product

Save Changes

Checkout

---

Secondary

Medium emphasis.

Used for supporting actions.

Examples

Cancel

View Details

Back

---

Outline

Low emphasis.

Used for optional actions.

Examples

Export

Preview

Duplicate

---

Ghost

Minimal emphasis.

Used inside tables and toolbars.

Examples

Edit

Filter

More

---

Danger

Destructive actions only.

Examples

Delete

Archive

Remove

# Loading State

Loading buttons

Disable interaction.

Replace icon with spinner.

Keep button width fixed.

# Disabled State

Disabled buttons

Reduce opacity.

Disable pointer interaction.

Maintain readable text.
