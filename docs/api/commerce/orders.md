# Orders API

Version: 1.0

Status: Frozen

Last Updated: July 2026

# Purpose

The Orders API manages customer purchases from checkout through fulfillment.

Orders are immutable financial records.

# Merchant Base Route

/api/v1/orders

# Storefront Checkout Route

/api/v1/store/{slug}/checkout

## Checkout

POST /api/v1/store/{slug}/checkout

Not Required

---

## Track Order

GET /api/v1/store/{slug}/orders/{orderNumber}

Authentication

Not Required

Description

Allows customers to track order progress using their order number.

# Business Rules

Orders cannot skip statuses.

Completed orders are immutable.

Cancelled orders remain in history.

Historical pricing never changes.

Order numbers are generated by the backend.

# Security

Merchants manage only their own orders.

Customers may only track their own order using the generated order reference.
