# ADR 0005

Title

Use NestJS for the Backend

Status

Accepted

Date

July 2026

# Context

Oja requires a structured backend that supports modular development, dependency injection, testing, and long-term maintainability.

# Decision

Use NestJS as the backend framework.

The backend follows

- Modular Architecture
- Dependency Injection
- Repository Pattern
- Service Layer

# Consequences

Positive

- Clear architecture
- Excellent TypeScript support
- Strong ecosystem
- Scalable codebase

Negative

- More boilerplate than lightweight frameworks

# Alternatives Considered

Express

Fastify

Hono

# Outcome

Accepted
